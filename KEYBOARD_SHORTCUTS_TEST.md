## 测试内容

### 1. Dashboard 页面快捷键

**测试步骤：**

1. 登录应用，进入 Dashboard 页面
2. 按 `Ctrl+N`（Windows/Linux）或 `Cmd+N`（Mac）
   - ✅ 预期结果：跳转到新建笔记页面 `/notes/new`
3. 返回 Dashboard，在搜索框外按 `Ctrl+F` / `Cmd+F`
   - ✅ 预期结果：搜索框获得焦点
4. 输入搜索关键词后，按 `Esc`
   - ✅ 预期结果：清除搜索，恢复显示所有笔记
5. 点击"选择"按钮进入多选模式，选择几条笔记后按 `Esc`
   - ✅ 预期结果：退出选择模式，清空选择状态

---

### 2. 新建笔记页面快捷键

**测试步骤：**

1. 在 Dashboard 按 `Ctrl+N` 进入新建笔记页面
2. 输入标题和内容
3. 按 `Ctrl+S` / `Cmd+S`
   - ✅ 预期结果：保存笔记并跳转到 Dashboard
   - ✅ 预期结果：不会触发浏览器默认的"保存网页"对话框
4. 再次进入新建笔记页面，输入一些内容后按 `Esc`
   - ✅ 预期结果：返回 Dashboard

---

### 3. 编辑笔记页面快捷键

**测试步骤：**

#### 查看模式

1. 在 Dashboard 点击任意笔记卡片，进入笔记详情页（查看模式）
2. 按 `Ctrl+E` / `Cmd+E`
   - ✅ 预期结果：切换到编辑模式
3. 按 `Esc`
   - ✅ 预期结果：返回 Dashboard

#### 编辑模式

1. 在查看模式按 `Ctrl+E` 进入编辑模式
2. 修改标题或内容
3. 按 `Ctrl+S` / `Cmd+S`
   - ✅ 预期结果：保存更改，切换回查看模式
   - ✅ 预期结果：不会触发浏览器默认的"保存网页"对话框
4. 再次进入编辑模式，修改内容后按 `Esc`
   - ✅ 预期结果：取消编辑，恢复原始内容，切换回查看模式

## 技术实现细节

### useKeyboardShortcuts Hook

位置：`src/hooks/useKeyboardShortcuts.ts`

**核心特性：**

- 自动识别操作系统（Mac vs Windows/Linux）
- 支持多个修饰键组合
- 可配置的 enabled 状态
- 可配置的 preventDefault 行为
- 自动清理事件监听器

**使用示例：**

```typescript
useKeyboardShortcuts([
  {
    key: "s",
    modifiers: { ctrl: true },
    callback: e => {
      e.preventDefault()
      handleSave()
    },
    enabled: isEditing, // 仅在编辑模式下启用
  },
])
```
