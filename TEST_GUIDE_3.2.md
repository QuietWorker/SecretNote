# 任务 3.2: 自动保存草稿功能 - 测试指南

## 功能概述

自动保存草稿功能会在用户输入时自动保存笔记内容到 localStorage，防止意外关闭页面导致数据丢失。主要特性包括：

- **防抖自动保存**：停止输入 2 秒后自动保存
- **草稿恢复**：页面重新打开时提示恢复草稿
- **视觉反馈**：显示保存状态（保存中、已保存、失败）
- **智能管理**：提交成功后自动清除草稿，24 小时后草稿过期

## 测试步骤

### 1. 新建笔记页面测试

#### 1.1 基本自动保存测试

1. 访问 http://localhost:3000/notes/new
2. 在标题输入框输入一些文字
3. 在内容输入框输入一些文字
4. **停止输入，等待 2 秒**
5. 观察右上角是否显示"已保存"提示（绿色对勾）
6. 打开浏览器开发者工具 → Application → Local Storage
7. 检查是否存在 `draft_note_new` 键
8. 确认其中包含你刚才输入的内容

#### 1.2 草稿恢复测试

1. 在新建笔记页面输入标题和内容
2. 等待自动保存（看到"已保存"提示）
3. **刷新页面**（F5 或 Ctrl+R）
4. 页面加载后，应该看到一个蓝色提示框："发现未保存的草稿，是否恢复？"
5. 点击"恢复"按钮
6. 确认表单中填充了之前输入的内容
7. 确认提示框消失

#### 1.3 放弃草稿测试

1. 重复 1.2 的步骤，直到出现草稿恢复提示
2. 点击"放弃"按钮
3. 确认表单被清空
4. 确认提示框消失
5. 检查 localStorage，确认 `draft_note_new` 已被删除

#### 1.4 提交后清除草稿测试

1. 在新建笔记页面输入标题和内容
2. 等待自动保存
3. 点击"保存"按钮提交笔记
4. 确认成功跳转到仪表板
5. 返回新建笔记页面
6. 确认**没有**出现草稿恢复提示
7. 检查 localStorage，确认 `draft_note_new` 已被删除

#### 1.5 防抖测试

1. 在标题输入框快速输入多个字符
2. 观察右上角状态
3. 确认**不会**每输入一个字符就保存一次
4. 只有在停止输入 2 秒后才会触发保存

## 常见问题

### Q1: 为什么有时候看不到自动保存提示？

A: 自动保存只在以下情况触发：

- 表单数据发生变化
- 停止输入 2 秒后
- 新建笔记页面：标题或内容不为空
- 编辑笔记页面：处于编辑模式

### Q2: 草稿会保存多久？

A: 草稿在 localStorage 中保存 24 小时。超过 24 小时后会自动清除。

### Q3: 自动保存会影响性能吗？

A: 不会。使用了 2 秒防抖，只有在用户停止输入后才会触发保存，不会频繁操作。

### Q4: 如果自动保存失败怎么办？

A:

- 会显示红色的"保存失败"提示
- 3 秒后提示消失
- 网络恢复后会自动重新保存
- 草稿仍然保存在 localStorage 中，不会丢失

### Q5: 提交表单时会自动保存吗？

A: 不会。提交表单时会暂停自动保存，防止冲突。提交成功后会清除草稿。
